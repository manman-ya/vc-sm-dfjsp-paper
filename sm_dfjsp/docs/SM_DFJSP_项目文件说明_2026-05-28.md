# SM-DFJSP / MVC-SM-DFJSP 项目文件说明

创建日期：2026-05-28  
适用目录：`sm_dfjsp/`

本文档用于快速定位项目文件。当前正式论文主线是 MVC-SM-DFJSP 两目标版本：`total_cost` 与 `makespan`。历史 SM-DFJSP 复现代码和历史实验结果仍保留，但正式论文数据、实验和图表应以统一固定成本口径重跑后的结果为准。

## 1. 顶层目录

| 路径 | 作用 |
| --- | --- |
| `pyproject.toml` | Python 项目配置、依赖和 pytest 配置。 |

| `scripts/` | 数据构造、实验运行、结果汇总和图表生成入口。 |
| `data/` | 原始 benchmark、扩展 MVC 数据集和中间数据。 |
| `reports/` | 实验输出、Pareto 点、指标、图件和 raw 结果。 |

| `tests/` | 单元测试和脚本 smoke 测试。 |
| `configs/` | 原 SM-DFJSP 复现实验配置。 |
| `基于共享制造的分布式柔性作业车间调度与供需匹配.pdf` | 原始共享制造 SM-DFJSP 参考论文。 |

## 7. 报告目录

`reports/` 保存实验结果。常见结构如下：

```text
reports/<experiment_name>/
  raw/       # 每次运行的 summary、solutions、checkpoint、selected 等
  pareto/    # 每实例、每算法、每 seed 的 Pareto CSV
  metrics/   # HV、IGD、GD、spacing、LB/UB 对比等
  figures/   # Pareto 图、甘特图、成本拆分图、负载图等
```

历史目录较多，正式论文使用前需要确认：

- 是否使用固定成本口径；
- 是否是两目标；
- 是否使用统一数据集；
- 是否包含所有算法和种子；
- 是否可由当前脚本复现。

## 9. 维护约定

- 新正式结果尽量新建 report 目录，不覆盖历史结果。
- 修改成本口径、目标函数或数据生成规则时，同步更新执行方案、实现清单和数据 README。
- 论文正文只引用统一口径后的数据和图表。
- `__pycache__/`、`.pytest_cache/` 和临时测试目录不作为论文产物。
- 若一个脚本既能跑历史口径又能跑正式口径，应在 `run_meta.json` 中写明成本定义、目标维度、数据源和参数。
