# EDA-TS 覆盖性整改说明

日期：2026-05-13

## 1. 整改背景

基于论文《Shared manufacturing-based distributed flexible job shop scheduling with supply-demand matching》中第 4 节 EDA-TS 算法描述和第 5.1 节性能指标定义，本次整改围绕覆盖性指标进行。论文中覆盖性主要由 IGD 和 C-metric 体现；非支配解记忆机制用于保存并利用当前非支配解，而不是累积大量重复目标点。

在 `reports/repro/sdmk05_full_run` 的结果中，EDA-TS 的原始 `nd_size` 明显大于唯一目标点数量，说明非支配解池存在重复目标点膨胀。重复点会影响三个方面：

1. 夸大 `nd_size`，使解集规模解释失真。
2. 重复目标点参与 PMA/PMS/PMM 更新时，会过度强化同一区域。
3. TS 从非支配池随机选择初始解时，容易反复搜索重复区域，降低有效覆盖。

## 2. 修改目标

本次修改目标如下：

1. 非支配解池按 `(cost, makespan)` 去重。
2. 非支配池超限时使用 crowding distance 保留分布更分散的点，而不是只保留低 cost 点。
3. TS 初始解从非支配池的稀疏区域中选择，避免重复区域被反复启动。
4. TS 邻域候选按非支配排序和拥挤距离选择，弱化原先按 `(cost, makespan)` 字典序选择导致的低 cost 偏置。
5. 实验导出同时记录 raw 和 unique 的 `nd_size` 口径，避免后续分析混淆。

## 3. 修改文件与内容

### 3.1 `src/smdfjsp/core/pareto.py`

修改内容：

1. 新增 `unique_by_objective(items)`：
   - 输入为 `(ObjPair, payload)` 列表。
   - 按目标值 `(cost, makespan)` 去重。
   - 保留第一次出现的代表个体。

2. 新增 `_truncate_by_crowding(items, max_size)`：
   - 用 `crowding_distance` 对非支配点进行分布保持。
   - 当非支配池超过 `max_size` 时，优先保留目标空间边界点和稀疏区域点。

3. 修改 `merge_non_dominated(...)`：
   - 合并 `base + incoming` 后先去重。
   - 非支配筛选后再次去重。
   - 超限时由原来的“按 `(cost, makespan)` 排序后保留最小 cost 前若干个”改为“按 crowding distance 截断”。

影响：

- EDA-TS、EDA-VNS、H-GA-TS、NSGA-II 等所有调用 `merge_non_dominated` 的算法都会返回唯一目标点前沿。
- `nd_pool` 不再被重复目标点膨胀。
- 池子超限时不再偏向低 cost 端，有利于保持 Pareto 前沿覆盖。

### 3.2 `src/smdfjsp/eda_ts/algorithm.py`

修改内容：

1. 新增 `_rank_by_front_and_crowding(objs)`：
   - 对候选目标值先做 fast non-dominated sorting。
   - 每个 front 内按 crowding distance 从大到小排序。
   - 用于覆盖性导向的候选排序。

2. 新增 `_select_ts_seeds(pool, count)`：
   - 从非支配池中选出 `Tmax` 个 TS 初始解。
   - 不再从 `nd_pool` 随机有放回抽样。
   - 优先选择目标空间中更稀疏的非支配点。

3. 修改 `_tabu_search(initial)`：
   - 增加 `local_nd`，记录 TS 搜索过程中遇到的局部非支配候选。
   - 邻域候选不再按 `(cost, makespan)` 字典序排序。
   - 改为对惩罚后的候选目标做非支配排序，再按 crowding distance 选择候选。
   - tabu aspiration 逻辑保留：若 tabu 候选支配当前最优解，仍允许通过。
   - TS 结束时优先返回局部非支配集合中 crowding distance 较大的代表解，以增强不同 TS 启动之间的覆盖差异。

4. 修改 `run()` 中 TS 启动逻辑：
   - 原逻辑：`for _ in range(self.cfg.tmax): random choice from ts_seed_pool`。
   - 新逻辑：`for seed_ind in self._select_ts_seeds(ts_seed_pool, self.cfg.tmax)`。

影响：

- TS 起点覆盖更均匀，减少重复目标点附近的反复搜索。
- TS 候选选择更接近论文中“fast non-dominated sorting + 非支配解记忆机制”的描述。
- 对低 cost 单方向的偏置降低，更有利于 IGD 和 C-metric。

### 3.3 `scripts/run_experiments_repeated.py`

修改内容：

1. 新增 `unique_objectives(front)`：
   - 对实验导出的目标点按 `(cost, makespan)` 去重。

2. 实验运行中同时保留：
   - `raw_fronts`：算法原始返回前沿。
   - `fronts`：按目标值去重后的前沿。

3. 指标计算改为基于 unique front：
   - `pf_true = build_pf_known(list(fronts.values()))`
   - `GD/IGD/C-metric` 均基于去重后的目标点计算。

4. `metrics_runs.csv` 增加字段：
   - `nd_size_raw`
   - `nd_size_unique`

5. 保留兼容字段：
   - `nd_size` 仍存在，但现在等同于 `nd_size_unique`。

6. `metrics_summary.csv` 增加字段：
   - `mean_nd_size_raw`
   - `mean_nd_size_unique`

影响：

- 后续结果表可以区分“原始解数量”和“唯一目标点数量”。
- 旧的 `mean_nd_size` 仍可读，但语义调整为唯一目标点均值。

### 3.4 `scripts/run_ablation_repeated.py`

修改内容：

1. 新增 `unique_front(front)`。
2. 消融实验中同时保留 `raw_fronts` 和去重后的 `fronts`。
3. GD、IGD、C-metric 改为基于去重后的目标点计算。
4. `ablation_runs.csv` 增加字段：
   - `nd_size_raw`
   - `nd_size_unique`
5. `ablation_summary.csv` 增加字段：
   - `mean_nd_size_raw`
   - `mean_nd_size_unique`

影响：

- 消融实验中的多种群、非支配记忆机制贡献会以唯一目标点口径进行评估。
- 避免重复点导致 EDA-TS 或其变体的 `nd_size` 被高估。

### 3.5 `tests/test_algorithm_regression.py`

修改内容：

1. 新增 `test_non_dominated_merge_deduplicates_objective_pairs`：
   - 验证 `merge_non_dominated` 会删除重复目标点。
   - 验证被支配目标点不会保留。

2. 修改 `test_edats_ablation_switches_produce_feasible_front`：
   - 增加断言：EDA-TS 及消融变体返回的 `nd_solutions` 中目标值不重复。

影响：

- 防止后续修改重新引入重复目标点膨胀问题。

## 5. 预期效果

预期变化如下：

1. `nd_size` 会明显下降，尤其是 EDA-TS 和 H-GA-TS。
2. `nd_size_unique` 更能反映真实 Pareto 覆盖规模。
3. EDA-TS 的 TS 启动点更分散，重复搜索减少。
4. C-metric 和 IGD 的解释更稳定，不再受重复目标点膨胀干扰。
5. GD 可能更真实地反映唯一前沿点质量；若唯一点更分散但局部收敛不足，GD 不一定立即下降。

## 6. 后续建议

1. 用相同配置重跑 `sdmk05` 的 `30 runs x 100s` 对比实验。
2. 对比整改前后的：
   - `mean_nd_size_raw`
   - `mean_nd_size_unique`
   - `mean_GD`
   - `mean_IGD`
   - `C(EDA-TS, EDA-VNS)`
3. 若 IGD 改善但 C-metric 仍弱于 EDA-VNS，再调整 MD/MC/MCT 的 top-k 随机化和参数组合。
4. 在 `sdmk05` 之外增加 SRU 异质性更强的实例验证，因为论文机制的优势更依赖共享制造中的供需匹配和 SRU 异质性。
