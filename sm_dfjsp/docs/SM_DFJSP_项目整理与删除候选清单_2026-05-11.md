## 2. 已删除内容

### 2.1 旧报告目录

已删除：

- `reports/repro/test_trace_tmp/`
- `reports/repro/x2_r3r4_sdmk05_full_n2_compare_only/`
- `reports/repro/sdmk05_default_full_n2_compare_only/`
- `reports/repro/single_instance_all_sdmk05/`
- `reports/repro/sdmk05_run/`

这些内容是什么：

- `test_trace_tmp/` 是单元测试生成的临时 trace 目录。
- 两个 `full_n2_compare_only/` 是 `n_runs=2` 的轻量 compare-only 产物，统计意义弱。
- `single_instance_all_sdmk05/` 与 `sdmk05_run/` 是早期 `sdmk05` 单实例报告，部分结果来自旧数据线或 quick 口径。

删除原因：

- 当前已有更接近论文口径的 `x2_r3r4_sdmk05_full_n10/`、`x2_r3r4_sdmk05_strict_run/` 和 `x2_r3r4_sdmk05_strict_validation/`。
- 旧报告容易和最新主线产物混淆。

### 2.2 旧阶段文档

已删除：

- `docs/SM_DFJSP_复现清单_除实例差异.md`
- `docs/SM_DFJSP_执行后完成清单_除实例差异.md`
- `docs/SM_DFJSP_mk01_15_复现计划.md`
- `docs/SM_DFJSP_论文对比文档.md`
- `docs/SM_DFJSP_完整复现执行文档.md`

这些内容是什么：

- 4 月阶段的计划、执行清单、早期论文对比和完整复现草案。
- 主要围绕旧 `sdmk01-15` 数据线和当时尚未进一步对齐的算法状态。

删除原因：

- 关键结论已经被 `ALGO_MAPPING.md`、`EDA_TS_复现说明文档.md`、`REPRO_REPORT.md`、当前进度文档和项目总览文档覆盖。
- 保留旧文档会增加口径冲突和维护成本。

### 2.3 旧数据线

已删除：

- `data/sdmk01-15/`

这部分是什么：

- 早期从 `mk01-15` 派生出的 SDMK 数据目录。
- 相比当前 `sdmk01-15_x2_r3r4/`，它不是最靠近论文共享制造口径的主线数据。

删除原因：

- 当前项目不再保留它作为旧基线。
- 后续数据主线统一到 `data/sdmk01-15_x2_r3r4/`。

### 2.4 Python 缓存文件

- Git 已跟踪的 `tests/**/__pycache__/`
